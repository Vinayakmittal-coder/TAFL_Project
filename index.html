§3 — Recognizing Automata

Computational Models

Each grammar class corresponds exactly to a computational model. More powerful automata recognize strictly larger language classes.

⊙
DFA / NFA
Finite Automaton
Finite set of states Q, input alphabet Σ, transition function δ: Q×Σ→Q. No auxiliary memory. Reads input left-to-right in O(n) time.
⊞
PDA
Pushdown Automaton
Finite automaton augmented with an unbounded stack Γ. Transitions may push/pop stack symbols. Handles nested structures (e.g. matched brackets).
⊡
LBA
Linear Bounded Automaton
Turing Machine restricted to tape length proportional to input: O(n) space. Handles context‑sensitive patterns like {aⁿbⁿcⁿ}. Decidable membership.
⊗
Turing Machine
Unrestricted TM
Infinite read/write tape, full set of transitions. Most powerful model of computation. Some languages in this class are undecidable (halting problem).
§4 — Decidability Properties

Decision Problems

"Yes" = decidable in finite time. "No" = undecidable. "Semi" = recognizable but may not halt on rejection.

§5 — Linguistic Perspective

Grammars as Mental Systems

Hunter, 2021
"The Chomsky hierarchy helped shift linguistics from viewing language as a collection of observed sentences to viewing it as a system of rules — a mental grammar — that generates sentences."

Intersubstitutability

Different expressions can be treated as equivalent in certain contexts, forming grammatical categories. The hierarchy levels correspond to different memory constraints on this tracking process.

Beyond String-Rewriting

When generating trees rather than strings, more restricted mechanisms (finite-state, strictly-local) can suffice to capture patterns relevant to syntactic theory.

Generative Capacity

The hierarchy explores what regularities can arise from grammars when specific structural constraints — restrictions on rules — are applied to them.

— T. Hunter, "The Chomsky Hierarchy." Blackwell Companion to Chomsky, 2021.
